# Система управления складом (Warehouse Management System)


Данный проект представляет собой реализацию REST-сервиса для управления складскими товарами в рамках учебной лабораторной работы. Сервис разработан с использованием Spring Boot и включает в себя многослойную архитектуру (Controller — Service — Repository), обработку HTTP-запросов с @RequestParam и @PathVariable, а также применение DTO для изоляции клиентской и внутренней моделей данных.

---
## Использованные технологии

- **Java 25**
- **Spring Boot / Spring Web**
- **Maven**
- **Postman** — тестирование API
- **Checkstyle, SonarCloud** — проверка качества кода

---

## Архитектура

Проект реализован из трёх основных уровней:

### ***1.Controller***
- обрабатывает HTTP-запросы
- Содержит методы с аннотациями @GetMapping, @PathVariable, @RequestParam
- Возвращает ответы в формате JSON
- Взаимодействует только с сервисным слоем
### ***2.Service***
- содержит бизнес-логику
- Выполняет преобразование Entity в DTO через маппер
- Получает данные от репозитория и передаёт контроллеру
- Независим от способа хранения данных
### ***3.Repository***
- хранит данные в памяти (список товаров)
- Предоставляет методы для получения и поиска товаров
- Инициализируется тестовыми данными при запуске

## Модель данных

### Сущность Товар
Класс Product *(товар)* представляет основную модель товара и содержит поля:
- `id` — уникальный идентификатор товара
- `name` — название товара
- `quantity` — количество на складе
- `price` — цена товара

### Data Transfer Object (DTO)
Для передачи данных клиенту используется ProductDto, который включает только необходимые поля:
**`id, name, price`**

Поле **`quantity`** скрыто от клиента, так как оно относится только к внутренней логике склада.

### Mapper
Класс `ProductMapper` отвечает за преобразование:

- `Product → ProductDto` — для ответа клиенту
- `ProductDto → Product` — для создания/обновления товара

Использование DTO и маппера позволяет отделить внутреннюю модель от внешнего API и контролировать формат ответа.

## REST-endpoints
- GET `/api/products` — возвращает список всех товаров на складе.

- GET `/api/products/{id}` — возвращает товар с указанным ID.

- GET `/api/products/search?query=` — ищет товары по названию .
